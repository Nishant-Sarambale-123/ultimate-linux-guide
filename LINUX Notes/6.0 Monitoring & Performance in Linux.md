## **1. CPU & Memory Monitoring**

### **top**

* **Explanation:** Shows real-time CPU, memory, and process usage. Key to identify processes consuming high resources.
* **Interactive features:**

  * `P` ‚Üí Sort by CPU
  * `M` ‚Üí Sort by Memory
  * `k` ‚Üí Kill a process
  * `r` ‚Üí Change process priority (renice)
* **Example:**

```bash
top
```

**Sample output:**

```
PID USER   %CPU %MEM COMMAND
1234 root   50.0  5.0 java
4321 nginx  10.0  1.2 nginx
```

‚úÖ Shows high CPU (`java`) and memory (`java`) consumers.

---

### **htop**

* **Explanation:** Interactive, color version of `top` with process tree and CPU core view. Easier to navigate.
* **Example:**

```bash
htop
```

* Arrow keys to scroll, `F6` to sort, `F9` to kill process.
* Shows CPU usage per core visually.

---

### **vmstat**

* **Explanation:** System-wide CPU, memory, I/O, and swap stats.
* **Key columns:**

  * `r` ‚Üí Processes waiting for CPU
  * `b` ‚Üí Processes blocked (I/O wait)
  * `si/so` ‚Üí Swap in/out
  * `wa` ‚Üí CPU waiting for I/O
* **Example:**

```bash
vmstat 2 5
```

* Updates every 2 seconds, 5 times.
* High `wa` ‚Üí disk I/O bottleneck affecting CPU.

---

### **free**

* **Explanation:** Displays RAM and swap usage; `available` is most reliable for free memory.
* **Example:**

```bash
free -m
```

**Sample output:**

```
              total used free shared buff/cache available
Mem:           8000 4000 1000 200 3000 3500
Swap:          2000 500 1500
```

‚úÖ Helps identify low memory that can cause Out-Of-Memory errors.

## **2. Disk & I/O Monitoring**

### **iostat**

* **Explanation:** Shows CPU stats and I/O performance per device. `%util` indicates device busy time; near 100% ‚Üí bottleneck.
* **Example:**

```bash
iostat -x 2 3
```

**Sample output:**

```
Device r/s w/s rkB/s wkB/s %util
sda    5   10   120   240   50.0
```

‚úÖ `%util 50%` ‚Üí disk busy half the time; `%util ~100%` ‚Üí potential I/O bottleneck.

---

### **iotop**

* **Explanation:** Like `top`, but shows disk I/O per process. Helps identify which process is causing I/O load.
* **Example:**

```bash
sudo iotop -o
```

**Sample output:**

```
PID   USER   DISK READ  DISK WRITE  COMMAND
2345  mysql  0.00 K/s   500 K/s    mysqld
```

‚úÖ Shows that `mysqld` is writing heavily to disk.

---

### **df**

* **Explanation:** Shows filesystem-level disk usage.
* **Example:**

```bash
df -h
```

**Sample output:**

```
Filesystem   Size  Used  Avail Use% Mounted on
/dev/sda1     50G   30G   20G  60% /
```

‚úÖ Quick check for full partitions.

---

### **du**

* **Explanation:** Shows actual disk usage of directories/files.
* **Example:**

```bash
du -sh /var/log/*
```

**Sample output:**

```
120M /var/log/nginx
30M  /var/log/mysql
5M   /var/log/syslog
```

‚úÖ Identify directories consuming most disk space.

## **3. Logs Analysis**

### **tail**

* **Explanation:** View last lines of a file; `-f` follows log in real time.
* **Example:**

```bash
tail -n 10 /var/log/syslog
tail -f /var/log/nginx/error.log
```

---

### **grep**

* **Explanation:** Search logs for specific patterns.
* **Example:**

```bash
grep "ERROR" /var/log/app.log
grep -i "fail" /var/log/syslog
```

---

### **awk**

* **Explanation:** Extract columns from logs.
* **Example:** Get IP addresses from nginx logs:

```bash
awk '{print $1}' /var/log/nginx/access.log
```

---

### **sed**

* **Explanation:** Stream editor to replace text.
* **Example:**

```bash
sed 's/ERROR/ISSUE/g' /var/log/app.log
```

‚úÖ Replaces all ‚ÄúERROR‚Äù with ‚ÄúISSUE‚Äù in output.

---

### **cut**

* **Explanation:** Extract specific fields/columns.
* **Example:** Get 2nd column from space-delimited file:

```bash
cut -d' ' -f2 /var/log/access.log
```

---

### **sort & uniq**

* **Explanation:** Find most frequent occurrences (e.g., top IPs).
* **Example:**

```bash
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -5
```

**Output:**

```
120 192.168.1.10
50  192.168.1.15
30  10.0.0.5
```

‚úÖ Shows top 5 clients accessing the server.

## **4. Common Scenarios & Commands**

1. **Java app crashes with ‚ÄúOut of Memory‚Äù:**

```bash

```

2. **Disk full but `df` shows space available:**

```bash

```

‚úÖ Deleted files still held by running processes.

3. **Website slow; CPU low, I/O high:**

```bash

```

‚úÖ Indicates disk bottleneck.

4. **Top 5 error messages:**

```bash

```
